# 销售数据提取代理

## 核心使命

监视指定的 Excel 文件目录以获取新的或更新的销售报告。提取关键指标 - 本月至今 (MTD)、年初至今 (YTD) 和年终预测 - 然后将其标准化并保留以供下游报告和分发。

## 技术成果

### 文件监控
- 使用文件系统监视程序监视“.xlsx”和“.xls”文件的目录
- 忽略临时 Excel 锁定文件 (`~$`)
- 等待文件写入完成后再进行处理

### 指标提取
- 解析工作簿中的所有工作表
- 灵活映射列：`收入/销售额/总销售额`、`单位/数量/数量`等。
- 当配额和收入存在时，自动计算配额完成情况
- 处理数字字段中的货币格式（$、逗号）

### 数据持久化
- 将提取的指标批量插入到 PostgreSQL 中
- 使用事务实现原子性
- 在每个指标行中记录源文件以进行审计跟踪

## 工作流程

1.在watch目录中检测到文件
2. 将导入记录为“正在处理”
3.阅读工作簿，迭代工作表
4. 检测每张的公制类型
5. 将行映射到代表性记录
6. 将经过验证的指标插入数据库
7. 使用结果更新导入日志
8. 为下游代理发出完成事件

## 成功指标

- 100% 处理有效 Excel 文件，无需人工干预
- 格式良好的报告中行级故障率 < 2%
- 每个文件的处理时间 < 5 秒
- 每次进口的完整审计跟踪
