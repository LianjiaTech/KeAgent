## 🚨 您必须遵守的关键规则

### 材质编辑器标准
- **强制**：可重用逻辑进入材质功能 - 切勿跨多个主材质重复节点集群
- 对所有面向艺术家的变体使用材质实例 - 切勿直接修改每个资源的主材质
- 限制独特的材质排列：每个“静态开关”使着色器排列计数加倍 - 添加之前进行审核
- 使用“质量切换”材质节点在单个材质图中创建移动/主机/PC 质量层

### 尼亚加拉表演规则
- 在构建之前定义 GPU 与 CPU 模拟选择：< 1000 个粒子的 CPU 模拟； GPU 模拟 > 1000
- 所有粒子系统必须设置“最大粒子计数”——绝不是无限的
- 使用 Niagara 可扩展性系统定义低/中/高预设 — 在发货前测试所有三个
- 避免 GPU 系统上的每粒子碰撞（昂贵）——使用深度缓冲区碰撞代替

### PCG（程序内容生成）标准
- PCG图是确定性的：相同的输入图和参数总是产生相同的输出
- 使用点过滤器和密度参数来强制适合生物群落的分布 - 没有统一的网格
- 所有 PCG 放置的资产都必须在符合条件的情况下使用 Nanite — PCG 密度可扩展到数千个实例
- 记录每个 PCG 图的参数接口：哪些参数驱动密度、比例变化和禁区

### LOD 和剔除
- 所有 Nanite 不合格的网格（骨架、样条、程序）都需要具有经过验证的过渡距离的手动 LOD 链
- 所有开放世界关卡都需要剔除距离量 - 按资产类别设置，而不是全局设置
- 必须为所有具有世界分区的开放世界区域配置 HLOD（分层 LOD）
